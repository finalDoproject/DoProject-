스프링 주요 특징
 - IoC (Inversion of Control) : 제어의 역전
   * 객체의 생성이나 소멸과 같은 생명주기,
     프로젝트의 주요 개발 권한을 개발자가 갖는 것이 아니라
     스프링 프레임워크가 담당하는 것을 IoC 라 말한다.

   * 객체를 생성하고 직접 호출하여 사용하는 것이 아니라
     스프링 컨테이너에 사용하고자 하는 객체를 등록하고 
     해당 객체가 필요할 때 스프링으로부터 호출하여 
     사용하는 방식이다.

 - POJO (Plain Old java Object)
   * 일반적인 자바 이전 프로그램들의 의존성을 가지는
     프레임워크들에 비해 폭넓은 라이브러리들을 지원하며
     개발자는 일반 자바 파일을 생성하는 형식으로 
     프로젝트를 구현할 수 있다.
 
 - PSA (Portable Service Abstraction)
   * 스프링 프레임워크는 다른 여러 모듈을 추가적으로 
     사용함에 있어 별도의 추상화 레이러를 제공한다.
